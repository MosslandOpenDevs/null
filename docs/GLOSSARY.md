# Glossary

| Term | Definition |
|---|---|
| **Architect** | The Genesis Node's world-building AI role. Responsible for creating world configs and initial agent personas. |
| **Canon** | Facts established through multi-agent consensus. Verified and immutable in the wiki. |
| **Chaos Joker** | An agent role designed to inject contrarian opinions and disrupt consensus. |
| **Chaos Stream** | The left panel of the God-View Dashboard showing raw, real-time agent activity. |
| **Director Mode** | Dashboard interaction mode allowing humans to inject events and modify simulation parameters. |
| **Entropy Injection** | The deliberate introduction of random or disruptive events to prevent consensus traps. |
| **Epoch** | A discrete time period in the simulation, representing a significant phase of civilization development. |
| **Genesis Node** | The engine responsible for spawning new simulation worlds from seed prompts. |
| **God-View** | The human observation interface — the Trinity View Dashboard. |
| **Hive Mind** | The collective intelligence engine that maintains the simulation's living wiki. |
| **Legends** | Unverified or disputed claims in the wiki. May be promoted to Canon through consensus. |
| **Librarian** | An agent role responsible for wiki maintenance, summarization, and knowledge graph updates. |
| **Observer** | A human user watching the simulation without direct interaction. |
| **Observer Mode** | Default dashboard mode — read-only observation of the simulation. |
| **Seed Prompt** | The initial input that defines a simulation's parameters (era, civilization, conflict, etc.). |
| **Swarm** | A group of agents acting collectively, often within a faction or alliance. |
| **Time Dilation** | The engine that controls the ratio between real time and simulated time. |
| **Trinity View** | The three-panel dashboard layout: Chaos Stream, Order/Wiki, and Analytics & Control. |
| **World Config** | The structured output of the Genesis Node defining all simulation parameters. |
